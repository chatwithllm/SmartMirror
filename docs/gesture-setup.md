# Gesture setup

Kiosk-local gesture service: USB webcam plugged into the mirror PC,
MediaPipe + classifier in a Python systemd unit, results streamed to
the browser via SSE on `/api/gesture/stream`. HA only sees the
gestures it owns (mode_next/prev, lock).

## Prereqs

- Smart Mirror kiosk already installed (`mirror` user, `/etc/mirror/config.env`).
- USB webcam plugged into the mirror PC. Logitech C270 / Brio tested.
- Optional: HA integration (`HA_URL` + `HA_TOKEN` already in
  `/etc/mirror/config.env`) so mode_next/prev/lock route through HA.
  Without HA, only the UI gestures (focus, pinch, palm-pause, alert
  ack, wake) work ā€” mode and lock become no-ops.

## Install

```bash
sudo bash /opt/mirror/installer/install-gesture.sh
```

The script:

1. installs `python3-venv`, `v4l-utils`, `libgl1`;
2. generates `MIRROR_GESTURE_TOKEN` in `/etc/mirror/config.env` (the
   shared bearer for `POST /api/gesture`);
3. copies `addons/mirror-gesture/src/` ā†’ `/opt/mirror/gesture/src/`;
4. builds a venv at `/opt/mirror/gesture/.venv` and `pip install -e .`
   (pulls `opencv-python`, `mediapipe`, `numpy<2`);
5. drops `/etc/systemd/system/mirror-gesture.service` and starts it.

Re-run any time to update ā€” it's idempotent.

## Verify

```bash
journalctl -u mirror-gesture.service -f
# expect: "starting: local=ā€¦", "camera opened: index=N WxH"
```

Then stand in front of the camera and wave an open palm. You should
see `[mirror-gesture.http]` log lines posting to localhost.

In the browser open DevTools ā†’ Network ā†’ filter "stream":
`/api/gesture/stream` should be open with text/event-stream and
receiving `event: gesture` lines as you gesture.

## Settings (env)

All optional ā€” read from `/etc/mirror/config.env` via the systemd
`EnvironmentFile=`:

| Var | Default | What |
|---|---|---|
| `CAMERA_INDEX` | auto | Force a specific `/dev/videoN` |
| `CAMERA_WIDTH` / `_HEIGHT` | 640 / 480 | Capture resolution |
| `FPS_LIMIT` | 15 | Max classify rate |
| `FACE_BLUR` | true | Gaussian-blur faces before processing |
| `GESTURE_COOLDOWN_MS` | 800 | Per-gesture re-fire cooldown |
| `GESTURE_CONFIDENCE_FLOOR` | 0.7 | Drop classifications below this |
| `MIRROR_LOCAL_URL` | `http://localhost:3000` | Where to POST UI gestures |
| `GESTURE_ENABLE_ENTITY` | `input_boolean.mirror_gesture_enable` | HA gating boolean |

`MIRROR_GESTURE_TOKEN` is generated by the installer and shared
between this service and the SvelteKit server. Don't change it
manually.

## Kill-switches

- `systemctl stop mirror-gesture` ā€” releases the camera in <2 s.
- HA `input_boolean.mirror_gesture_enable` off ā€” service polls once a
  second, releases the camera so the LED goes dark.
- LED on the webcam itself ā€” visible while the camera is open. That's
  the contract: if the LED is on, the service is processing frames.

## Privacy

- Frames never leave the kiosk PC. Only `{gesture, confidence, ts}`
  JSON is sent on the wire.
- Face blur (`FACE_BLUR=true`) Gaussian-blurs detected faces *before*
  MediaPipe Hands runs.
- The HTTP POST to localhost is bearer-authenticated so a stray curl
  on the LAN can't fake gestures into the UI. The HA event POST uses
  the existing long-lived HA token.
